# Uclaw 需求验证调研方案

> **项目背景**：产品经理参考 ShopClawMart 业态，计划在 Compshare 下做一个类似平台「Uclaw」——AI Agent 技能/角色的交易市场。需要快速验证存量客户是否有此需求。
> 
> **调研目标**：用最短时间验证三个核心假设，拿到 Go/No-Go 决策依据。
> 
> **时间要求**：总周期 **5 个工作日**，出结果。

---

## 一、三个核心假设

| # | 假设 | 验证标准（Go 的门槛） |
|---|------|----------------------|
| H1 | Compshare 存量客户中有相当比例在使用或关注 AI Agent | ≥40% 受访者正在使用或计划使用 AI Agent |
| H2 | 客户愿意为「开箱即用」的 Agent 技能/角色付费 | ≥25% 受访者表示愿意付费 |
| H3 | 客户认为在 Compshare 生态内提供此服务有吸引力（GPU+模型+Agent 一站式） | ≥30% 受访者认为"有吸引力"或"非常有吸引力" |

**三个假设全过 → Go；两个过 → 有条件 Go（需调整方向）；不到两个 → No-Go 或 Pivot**

---

## 二、调研设计：三板斧

### 第一板斧：问卷筛选（Day 1-2）

**目标**：广撒网，量化需求面，筛出深访对象

**渠道**（按优先级）：
1. Compshare 控制台内嵌弹窗（登录用户直接触达）
2. 已有客户邮件列表（历史充值用户）
3. Compshare 微信社群/钉钉群

**样本量目标**：≥100 份有效问卷

**问卷设计（控制在 2 分钟内完成，共 8 题）**：

```
Q1. 你目前使用 Compshare 的主要场景是？（单选）
   □ 模型训练/微调
   □ 模型推理/部署
   □ API 调用（Modelverse）
   □ 数据处理
   □ 其他：____

Q2. 你目前是否在使用 AI Agent（如 Claude/ChatGPT 的自动化助手、AutoGPT、OpenClaw 等）？（单选）
   □ 正在日常使用
   □ 试用过但没持续用
   □ 听说过但没用过
   □ 不了解

Q3. 你在使用 AI Agent 时，最大的痛点是什么？（多选，最多 3 项）
   □ 配置太复杂，不知道怎么调
   □ 找不到适合自己业务的 Agent/技能
   □ 质量参差不齐，试错成本高
   □ 没有中文/本土化的好方案
   □ 缺少和现有工具的集成
   □ 没有痛点 / 不适用

Q4. 如果有一个平台提供「开箱即用」的 AI Agent 技能包（类似应用商店），你会感兴趣吗？（单选）
   □ 非常感兴趣，愿意付费购买
   □ 比较感兴趣，免费的话会用
   □ 一般，看具体内容
   □ 不感兴趣

Q5. 你最希望这类技能包覆盖哪些场景？（多选，最多 3 项）
   □ 代码生成/审查/DevOps
   □ 数据分析/报表
   □ 内容创作/营销
   □ 客服/知识库
   □ 行业研究/竞品分析
   □ 其他：____

Q6. 你能接受的单个技能包价格区间是？（单选）
   □ 免费才用
   □ ¥10 以内
   □ ¥10-50
   □ ¥50-200
   □ ¥200 以上（企业级）

Q7. 如果 Compshare 推出此类服务（GPU 算力 + 模型 API + Agent 市场一站式），对你的吸引力如何？（单选）
   □ 非常有吸引力（一站式省心）
   □ 比较有吸引力
   □ 一般
   □ 没什么吸引力

Q8. 你愿意参加 15 分钟的深度访谈吗？（完成可获 ¥50 Compshare 算力券）
   □ 愿意（请留联系方式：____）
   □ 不愿意
```

**数据分析重点**：
- Q2 → 验证 H1（Agent 使用渗透率）
- Q4 + Q6 → 验证 H2（付费意愿 + 价格敏感度）
- Q7 → 验证 H3（一站式吸引力）
- Q3 + Q5 → 辅助判断产品方向（痛点 + 场景优先级）

---

### 第二板斧：快速深访（Day 3-4）

**目标**：定性挖掘需求深度，理解场景和决策逻辑

**对象**：从问卷 Q8 中筛选 **6-8 人**，按以下分层抽样：

| 用户类型 | 人数 | 筛选条件 |
|---------|------|---------|
| 重度 Agent 用户 | 2-3 人 | Q2="正在日常使用" + 月消费较高 |
| 轻度/观望用户 | 2-3 人 | Q2="试用过" 或 "听说过" |
| 高付费意愿用户 | 2 人 | Q4="非常感兴趣" + Q6≥"¥10-50" |

**访谈提纲（15 分钟，半结构化）**：

```
【暖场 - 1 min】
"感谢参加，聊 15 分钟就好。主要想了解你用 AI Agent 的情况。"

【当前状态 - 5 min】
1. 你现在用哪些 AI Agent/自动化工具？用来做什么？
2. 这些工具的配置和上手过程顺利吗？花了多长时间？
3. 有没有想实现但没做成的 Agent 场景？卡在哪里了？

【需求探索 - 5 min】
4. 如果有人帮你做好了一个 [他提到的场景] 的 Agent 技能包，你会买吗？
   → 追问：你判断值不值得买的标准是什么？
5. 你更希望买「完整角色」（如一个 AI 数据分析师）还是「单项技能」（如一个 SQL 生成器）？
   → 追问：为什么？
6. 如果这个市场放在 Compshare 里，你觉得相比独立平台有优势吗？

【收尾 - 2 min】
7. 你觉得这种产品最不能缺的一个功能是什么？
8. 还有什么想补充的？
```

**访谈输出**：每人一页访谈摘要，包含：
- 用户画像（角色/公司/使用场景）
- 关键 quotes（原话）
- 需求强度评级（强需求 / 弱需求 / 无需求）
- 对 Uclaw 的态度和顾虑

---

### 第三板斧：数据交叉验证（Day 4-5）

**目标**：用 Compshare 后台数据补充问卷/访谈的盲区

**数据分析清单**（需产品/数据团队配合）：

| 分析项 | 数据来源 | 找什么 |
|-------|---------|-------|
| Modelverse API 调用量趋势 | API 日志 | Agent 相关模型（Claude/GPT）调用在增长吗？ |
| 社区镜像中 Agent 类镜像使用情况 | 镜像使用数据 | 有多少人在用 AutoGPT/OpenClaw 等 Agent 镜像？ |
| 客户工单中 Agent 相关问题 | 工单系统 | 有没有客户问"怎么搭 Agent"？频率如何？ |
| 客户行业分布 × Agent 渗透率 | 用户画像 + Q2 交叉 | 哪些行业客户 Agent 需求最强？ |

| Day | 动作 | 输出物 |

## 五、资源需求

| 资源 | 说明 | 负责方 |
|------|------|--------|
| 问卷工具 | 腾讯问卷/金数据（免费版即可） | 用研 |
| 控制台弹窗 | 需前端配合嵌入问卷入口 | 产品+前端 |
| 客户邮件列表 | 历史充值用户列表 | 运营 |
| 访谈激励 | ¥50 算力券 × 8 人 = ¥400 | 运营审批 |
| 后台数据提取 | API 日志 + 镜像使用数据 | 数据团队 |
| 深访执行 | 电话/腾讯会议，15 min/人 | 用研 |

**总成本**：约 ¥400 算力券 + 5 天人力

---

## 六、风险控制

| 风险 | 应对 |
|------|------|
| 问卷回收量不够（<100） | Day 2 晚追加社群推送 + 小额激励（¥5 算力券） |
| 深访招募不到人 | 降低门槛：电话访谈 10 分钟，提高激励到 ¥80 |
| 客户都说"感兴趣"但实际不付费 | Q6 的价格锚定 + 深访中用"你上次为类似东西花了多少钱"验证 |
| 数据团队排不上 | 先用问卷+访谈做决策，数据验证作为补充异步进行 |
