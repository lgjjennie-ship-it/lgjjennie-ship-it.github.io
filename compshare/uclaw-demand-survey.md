# Uclaw 需求验证调研方案（v2）

> **项目背景**：产品经理参考 ShopClawMart 业态，计划在 Compshare 下做一个类似平台「Uclaw」——AI Agent 技能/角色的交易市场。需要快速验证存量客户是否有此需求。
> 
> **调研目标**：通过了解客户**当下真实行为**（而非未来意愿），判断市场是否存在。
> 
> **核心原则**：不问"你想不想买"，只问"你现在在做什么、花了多少、踩了什么坑"。
> 
> **时间要求**：总周期 **5 个工作日**，出结果。

---

## 一、调研逻辑：从行为推需求

传统问法（❌）：你会不会买 Uclaw 的产品？
→ 得到的是**态度数据**，用户会高估自己的付费意愿

正确问法（✅）：你现在在用什么？花了多少钱？为什么花？哪里不爽？
→ 得到的是**行为数据**，用真实消费行为反推需求强度

**推导链条**：
```
当下行为 → 已验证需求 → 现有方案的痛点 → Uclaw 的切入机会
```

---

## 二、验证框架：四个关键问题

| # | 要回答的问题 | 判断标准 |
|---|-------------|---------|
| Q1 | 客户现在是否在用 AI Agent？渗透率多高？ | ≥30% 在用 → 市场存在 |
| Q2 | 在用的客户为此花了多少钱？花在哪里？ | 有实际付费行为 → 需求真实 |
| Q3 | 客户在 token 成本控制上做了什么？ | 有主动优化行为 → 痛点真实 |
| Q4 | 没在用的客户，卡在哪里？是没需求还是有障碍？ | 有障碍 → 有机会；没需求 → 放弃 |

**Q1+Q2 验证"需求是否存在"，Q3+Q4 验证"痛点是否成立"**

---

## 三、问卷设计（控制在 3 分钟内，共 10 题）

问卷采用**分支逻辑**：Q2 回答决定后续走哪条路径。

### 通用部分（所有人回答）

```
Q1. 你目前使用 Compshare 的主要场景是？（多选）
   □ 模型训练/微调
   □ 模型推理/部署
   □ API 调用（Modelverse）
   □ 数据处理/分析
   □ 其他：____

Q2. 你目前是否在使用 AI Agent 类工具？
   （Agent 指：能自主完成多步骤任务的 AI 工具，如 OpenClaw、AutoGPT、
    Dify、Coze/扣子、Claude MCP、自建 Agent 等）
   □ A. 日常在用（每周至少用几次）→ 走【在用路径】
   □ B. 试过但没持续用          → 走【弃用路径】
   □ C. 了解但没用过            → 走【观望路径】
   □ D. 不太了解                → 走【结束】
```

### 在用路径（Q2=A，深挖付费行为和成本控制）

```
Q3. 你目前在用哪些 Agent 工具/平台？（多选）
   □ OpenClaw（Claude/GPT Agent）
   □ Dify
   □ Coze / 扣子（字节）
   □ AutoGPT / AgentGPT
   □ LangChain / LlamaIndex 自建
   □ Claude MCP
   □ 企业内部自研
   □ 其他：____

Q4. 你为 Agent 相关工具/服务每月大概花多少钱？（含 API 调用费、订阅费等）
   □ 没花钱（用免费方案）
   □ ¥100 以内
   □ ¥100-500
   □ ¥500-2000
   □ ¥2000-10000
   □ ¥10000 以上

Q5. 这笔钱主要花在哪里？（多选，最多 3 项）
   □ 模型 API 调用费（token 费用）
   □ Agent 平台订阅费（如 Dify Pro、Coze 会员等）
   □ 购买现成的 Agent 技能/插件/模板
   □ 开发人力成本（自建 Agent）
   □ GPU 算力（跑本地模型降低 API 成本）
   □ 其他：____

Q6. 为了控制 Agent 的 token/API 成本，你采取过哪些手段？（多选）
   □ 切换到更便宜的模型（如从 GPT-4 切到 DeepSeek/Qwen）
   □ 优化 prompt 减少 token 消耗
   □ 本地部署开源模型替代 API
   □ 设置调用频率/额度上限
   □ 用缓存/RAG 减少重复调用
   □ 没刻意控制过
   □ 其他：____

Q7. 你目前用 Agent 最大的痛点是什么？（多选，最多 2 项）
   □ 配置/调试太耗时间
   □ token 成本太高
   □ 效果不稳定，输出质量波动大
   □ 缺少适合我场景的现成方案
   □ 和现有工作流集成困难
   □ 数据安全/隐私顾虑
   □ 其他：____
```

### 弃用路径（Q2=B，了解为什么放弃）

```
Q8. 你试用过哪些 Agent 工具后放弃了？为什么？（开放题，一两句话即可）
   工具名称：____
   放弃原因：____

Q9. 如果以下问题被解决，你会重新考虑使用吗？（多选）
   □ 有人帮我配好，开箱即用
   □ 成本降到可接受范围
   □ 有适合我行业/场景的专用方案
   □ 效果更稳定可靠
   □ 不会重新考虑，当前不需要
```

### 观望路径（Q2=C，了解是没需求还是有障碍）

```
Q8c. 你了解但没用 Agent 的主要原因是？（单选）
   □ 不知道能用来做什么（不确定价值）
   □ 配置门槛太高（不会搞）
   □ 成本顾虑
   □ 现有工具够用，不需要
   □ 数据安全顾虑
   □ 其他：____
```

### 收尾（所有人）

```
Q10. 你愿意参加 15 分钟的电话访谈吗？（完成可获 ¥50 Compshare 算力券）
   □ 愿意（请留联系方式：____）
   □ 不愿意
```

## 四、深度访谈提纲（Day 3-4，6-8 人）

### 抽样策略

| 用户类型 | 人数 | 筛选条件 | 访谈重点 |
|---------|------|---------|---------|
| 高消费 Agent 用户 | 2-3 人 | Q4≥¥500/月 | 花钱花在哪？为什么值得？ |
| 有成本控制行为的用户 | 2 人 | Q6 选了 ≥2 项 | 控制成本的具体做法和效果 |
| 弃用者 | 1-2 人 | Q2=B | 到底卡在哪？什么能让你回来？ |
| 有障碍的观望者 | 1 人 | Q8c≠"不需要" | 障碍到底是什么？ |

### 访谈提纲（15 分钟，半结构化）

**对"在用"的用户：**
```
1. 你用 Agent 主要做什么事？能举一个具体例子吗？
2. 你现在的方案是自己搭的还是用现成的？花了多久搭起来？
3. 每个月在这上面花多少钱？你觉得值吗？为什么？
4. 你有没有试过买别人做好的 Agent 模板/技能？在哪买的？体验如何？
5. 如果要降低 token 成本，你试过什么办法？效果怎么样？
6. 你现在最想解决但还没解决的问题是什么？
```

**对"弃用"的用户：**
```
1. 你当时用 Agent 想解决什么问题？
2. 最后为什么放弃了？能说说具体经历吗？
3. 你现在用什么方式在做当时想用 Agent 做的事？
4. 如果有人帮你把 Agent 配好了，你直接用，这样你会考虑吗？
```

**对"观望"的用户：**
```
1. 你在哪里了解到 AI Agent 的？
2. 你觉得 Agent 能帮你做什么？
3. 是什么让你一直没上手？（追问具体障碍）
4. 你现在有没有什么重复性工作觉得应该自动化但还没做到的？
```

## 五、数据分析框架

### 问卷分析重点

| 分析维度 | 数据来源 | 关键指标 |
|---------|---------|---------|
| **市场渗透率** | Q2 分布 | 在用比例、弃用比例、观望比例 |
| **付费现状** | Q4 + Q5 | 月均花费、钱花在哪里 |
| **成本敏感度** | Q6 | 有多少人在主动控制成本、用什么方法 |
| **核心痛点** | Q7 / Q8 / Q8c | 痛点排名、弃用原因、观望障碍 |
| **场景需求** | Q1 × Q3 交叉 | 哪类 Compshare 用户更可能用 Agent |

### Go/No-Go 判断矩阵

```
                    付费行为真实（Q4 有实际花费）
                    是              否
渗透率高    ┌──────────────┬──────────────┐
(Q2≥30%在用)│   强 Go      │  有条件 Go   │
            │ 有人有需求   │ 有人但没找到  │
            │ 且在花钱     │ 付费方式      │
            ├──────────────┼──────────────┤
渗透率低    │   Niche Go   │   No-Go      │
(Q2<30%在用)│ 小众但高价值 │  市场不存在   │
            │ 做垂直切入   │  或时机未到   │
            └──────────────┴──────────────┘
```

### 交叉验证：后台数据

| 分析项 | 数据来源 | 验证什么 |
|-------|---------|---------|
| Modelverse API 中 Agent 相关模型调用量 | API 日志 | 和问卷 Q2 的渗透率是否吻合 |
| 长上下文/多轮对话 API 调用占比 | API 日志 | Agent 典型特征是多轮+长上下文 |
| 客户工单中 Agent/自动化相关问题 | 工单系统 | 是否有人在主动问"怎么搭 Agent" |

## 六、时间线与输出物

| Day | 动作 | 输出物 |
|-----|------|--------|
| Day 1 | 问卷上线 + 控制台弹窗 + 邮件/群推送 | 问卷链接 |
| Day 2 | 问卷回收 + 数据统计 | 问卷数据报告（渗透率、付费现状、痛点排名） |
| Day 3 | 深访 4 人（高消费 + 成本控制型） | 4 份访谈摘要 |
| Day 4 | 深访 2-4 人（弃用者 + 观望者）+ 后台数据 | 访谈摘要 + 数据验证 |
| Day 5 | 汇总分析 | **《Uclaw 需求验证报告 v2》** |

### 最终报告结构

```
1. 执行摘要（一页纸结论）
2. 市场现状：Agent 渗透率 + 付费规模
3. 客户画像：谁在用、用什么、花多少
4. 成本痛点：控制 token 的手段和效果
5. 弃用/观望分析：障碍在哪里
6. 机会判断：Go/No-Go + 切入方向建议
7. 附录：原始数据 + 访谈记录
```
